## 🔧 Instalación y Configuración

### 1. Crear entorno virtual
```bash
python -m venv venv
source venv/bin/activate  # En Windows: venv\Scripts\activate
```

### 2. Instalar dependencias
```bash
pip install -r requirements.txt
```

### 3. Configurar variables de entorno (.env)
```env
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=tu_password
DB_NAME=control_acceso
SECRET_KEY=tu_secret_key_super_segura
JWT_SECRET_KEY=jwt_secret_key_diferente
FLASK_ENV=development
```

### 4. Inicializar base de datos
```bash
python -c "from utils.database import init_db; init_db()"
```

### 5. Ejecutar la aplicación
```bash
python app.py
```

## 📋 Endpoints de la API

### Autenticación
- `POST /api/auth/login` - Iniciar sesión
- `POST /api/auth/logout` - Cerrar sesión
- `POST /api/auth/refresh` - Renovar token

### Usuarios
- `GET /api/usuarios` - Listar usuarios
- `POST /api/usuarios` - Crear usuario
- `GET /api/usuarios/{id}` - Obtener usuario
- `PUT /api/usuarios/{id}` - Actualizar usuario
- `DELETE /api/usuarios/{id}` - Eliminar usuario

### Accesos
- `POST /api/accesos/registrar` - Registrar entrada/salida
- `GET /api/accesos/usuario/{id}` - Historial de usuario
- `GET /api/accesos/hoy` - Accesos del día
- `GET /api/accesos/rango` - Accesos por rango de fechas

### Reportes
- `GET /api/reportes/asistencia` - Reporte de asistencia
- `GET /api/reportes/tardanzas` - Reporte de tardanzas
- `GET /api/reportes/ausencias` - Reporte de ausencias
- `GET /api/reportes/dashboard` - Datos para dashboard

### Administración
- `GET /api/admin/dispositivos` - Gestión de dispositivos
- `POST /api/admin/configuracion` - Configuración del sistema
- `GET /api/admin/logs` - Logs del sistema

## 🔐 Seguridad

- **JWT Tokens** para autenticación
- **Bcrypt** para hashear contraseñas
- **Rate Limiting** en endpoints críticos
- **Validación** de datos de entrada
- **CORS** configurado apropiadamente
- **Logs** de auditoría para todas las acciones
